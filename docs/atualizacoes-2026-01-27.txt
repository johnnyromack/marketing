1. AUTENTICAÇÃO COM GOOGLE
================================================================================

--- Implementação do Login com Google ---
Arquivo: src/pages/Auth.tsx

Funcionalidades:
- Botão de login com Google com ícone oficial
- Função handleGoogleSignIn para autenticação OAuth
- Estado de carregamento isGoogleLoading
- Redirecionamento automático após autenticação
- Separador visual "ou continue com email"

Código implementado:
```typescript
const handleGoogleSignIn = async () => {
  setIsGoogleLoading(true);
  const { error } = await supabase.auth.signInWithOAuth({
    provider: 'google',
    options: {
      redirectTo: `${window.location.origin}/`,
    },
  });
  setIsGoogleLoading(false);
  if (error) {
    toast({
      title: 'Erro ao entrar com Google',
      description: error.message,
      variant: 'destructive',
    });
  }
};
```

Configuração necessária no Google Cloud Console:
- Adicionar URLs de redirect autorizadas:
  * https://nzmherisqehvzhmsxmeb.supabase.co/auth/v1/callback
  * https://publicidaderaiz.lovable.app

2. MELHORIAS DE UX NA PÁGINA DE LOGIN
================================================================================

--- Toggle de Mostrar/Ocultar Senha ---
Arquivo: src/pages/Auth.tsx

Funcionalidades:
- Ícone Eye/EyeOff para alternar visibilidade da senha
- Estado showPassword para controlar visibilidade
- Botão não focável (tabIndex={-1}) para melhor navegação

Código implementado:
```typescript
const [showPassword, setShowPassword] = useState(false);

<Input
  type={showPassword ? "text" : "password"}
  className="pl-10 pr-10"
/>
<button
  type="button"
  onClick={() => setShowPassword(!showPassword)}
  className="absolute right-3 top-1/2 -translate-y-1/2"
  tabIndex={-1}
>
  {showPassword ? <EyeOff /> : <Eye />}
</button>
```

3. CORREÇÕES DE WARNINGS DO REACT
================================================================================

--- Correção de forwardRef ---
Arquivos: 
- src/components/RaizLogo.tsx
- src/components/ThemeLogo.tsx

Problema:
- Warnings no console sobre componentes que recebiam ref sem usar forwardRef

Solução aplicada:
```typescript
// Antes
export function RaizLogo({ className, alt }: RaizLogoProps) {
  return <img src={logoSrc} alt={alt} className={className} />;
}

// Depois
export const RaizLogo = forwardRef<HTMLImageElement, RaizLogoProps>(
  ({ className = "h-8 w-auto", alt = "Raiz Educação" }, ref) => {
    // ... lógica do componente
    return <img ref={ref} src={logoSrc} alt={alt} className={className} />;
  }
);
RaizLogo.displayName = 'RaizLogo';
```

Componentes atualizados:
- RaizLogo: Logo da Raiz Educação com troca automática dark/light
- ThemeLogo: Logo da RomackVision com troca automática dark/light
